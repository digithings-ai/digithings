DigiGraph

The orchestration brain. A LangGraph supervisor with a sub-graph registry routes every request through the right specialist agents, with structured Pydantic v2 outputs and LangGraph checkpointing throughout.

  • MCP-based tool registry: builtins plus vertical capabilities exposed via POST /v1/orchestrator_tools.
  • LiteLLM routing with caching; pick your model tier with DIGI_LLM_MODE.
  • OpenAI-compatible surface so any existing SDK can drive it.

DigiQuant

Quantitative strategy engine powered by NautilusTrader — a Rust core for ultra-low-latency backtest, optimize, and execute. The flagship vertical, and the highest quality bar on the stack.

  • Atlas — high-level fundamental and research engine feeding strategy biases.
  • Hermes — delivery layer: deliberation pipelines, delta updates, and signal publishing.
  • Kairos — timing and execution layer; loopback-only by default, human gates before any live trade.

DigiSearch

Retrieval-Augmented Generation and document search, built on Polars for massive-scale ingest without pandas overhead. Selective indexing rules keep relevance high and cost low.

  • Pluggable vector store registry: pgvector, Qdrant, in-memory, and more.
  • Multi-mode retrieval — dense, sparse, hybrid — selected per query.
  • Entity-neutral API: Document, Chunk, Query, Result.

DigiChat

Production Next.js chat UI and BFF for DigiGraph. Deployed at chat.digithings.ai, it adapts to whatever scopes your JWT grants and supports bring-your-own-key against any LiteLLM-routed provider.

  • Auth.js + Drizzle, with scoped API keys from DigiKey.
  • AI SDK streaming, structured tool calls, embedded surfaces.
  • BYOK settings panel — keys stay client-side.

DigiKey

The auth plane for every service-to-service and user-to-service call. RS256 JWTs with a JWKS endpoint, scoped API keys, SSO identity federation, and an org + project membership model.

  • Row-level resource access baked into issued tokens.
  • Scoped keys for CI, agents, and end users.
  • Runs standalone or embedded; consumed by every other module.

DigiSmith

Observability across the stack. Correlation IDs propagate via X-Request-ID, spans carry workflow_id, request_id, and session_id, and PII is redacted before anything is written.

  • Prometheus /metrics with version and environment labels.
  • Public /v1/status endpoint — secret-free by contract.
  • Structured logging helpers every service imports.

DigiClaw

The always-on agent runtime. Heartbeats, immutable JSONL audit, and an MCP skill surface make long-running autonomous work safe and observable. Atlas runner scheduling and drift detection live here.

  • Strict loopback-only by default; human gates for anything consequential.
  • Audit via digibase.audit.redact_mapping — no raw prompts, keys, or document bodies persisted.
  • Deployment gateway for 24/7 agent work.

DigiBase

Shared Python library, deliberately minimal. Not a service — a common foundation of HTTP middleware, audit helpers, and error conventions so every other module behaves consistently.

  • Request-ID middleware and ContextVar-based log correlation.
  • Redacted audit logging used everywhere.
  • Future home of the multi-tenant credential broker.

DigiStore

Storage abstraction layer. One typed API over S3, MinIO, Postgres, and SQLite — so strategy artifacts, research outputs, and vector payloads travel the same path no matter where they land.

  • Multi-backend: swap S3 for MinIO or SQLite without touching business code.
  • Works alongside OpenBB-style data providers for financial datasets.
  • Single source of truth for blob, row, and vector storage contracts.

DigiLink

The protocol translation layer. When an external system speaks something DigiGraph doesn't — a proprietary broker feed, a legacy REST surface, an odd webhook — DigiLink adapts it into MCP tool calls the orchestrator already understands.

  • Built for bridging non-MCP transports cleanly.
  • Keeps the orchestrator's tool surface uniform.
  • Adapters are small, testable, and composable.
