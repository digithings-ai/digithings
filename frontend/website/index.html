Act I — Orchestration

DigiGraph

The brain. A LangGraph supervisor routes every request.

A declarative tool registry (OpenAI-compatible function schemas) pairs with a sub-graph supervisor. Any vertical can expose its capabilities via POST /v1/orchestrator_tools and drop straight in.

LiteLLM routes calls; LangGraph checkpoints keep state durable across agent hops. The whole thing speaks the OpenAI API on the front door.

Act II — Quantitative

DigiQuant

The flagship vertical. Research, execute, audit.

NautilusTrader core with a strategy registry: name your strategy, bind a config, pick defaults. Atlas reasons over research, Hermes publishes signals, Kairos executes — loopback-only, human-gated.

Act III — Retrieval

DigiSearch

Ingest. Chunk. Embed. Search. Polars throughout.

A single DigiSearch client resolves index backends on demand — Chroma, pgvector, Qdrant, or in-memory. Entity names are backend-neutral: Document, Chunk, Query, Result.

Selective indexing rules keep relevance high and cost low; multi-mode retrieval (dense / sparse / hybrid) is chosen per query.

Act IV — Chat

DigiChat

Next.js chat UI + BFF. Streams DigiGraph. BYOK.

Machine API keys or a user session — DigiChat brokers DigiGraph traffic with the right bearer, tenant, and LiteLLM proxy key. The UI adapts to whatever scopes your JWT grants.
