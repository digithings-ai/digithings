Act I · Orchestration

DigiGraph

One supervisor decides which specialist runs. Every time.

Single-agent setups break the moment a request needs more than one tool path. DigiGraph's LangGraph supervisor inspects each request and routes to the right sub-graph — quant research, vector retrieval, chat — with a declarative tool registry that any new vertical can plug into via POST /v1/orchestrator_tools.

Front door speaks the OpenAI API, so existing clients work unchanged. LiteLLM handles model routing and caching; LangGraph checkpoints persist state across hops, so a multi-step run survives a restart.

Act II · Quantitative

DigiQuant

Strategy research that ends in an order, not a markdown file.

Atlas runs scheduled research cycles, Hermes turns the output into signals, Kairos executes them on a NautilusTrader core. Every step writes to an immutable audit trail; live trading stays loopback-only until a human flips the gate. No black-box cloud platform. No vendor-priced data feed required.

Act III · Retrieval

DigiSearch

Production RAG without a stack rewrite when you switch vector DB.

One DigiSearch client over Chroma, pgvector, Qdrant, or in-memory — backend-neutral Document, Chunk, Query, and Result entities mean you swap engines without touching business code.

Dense, sparse, and hybrid retrieval are first-class — pick per query, not per index. Polars throughout: ingest pipelines stay fast on a laptop, scale to a server without rewriting in pandas.

Act IV · Chat

DigiChat

Talk to your stack with your keys, your models, your audit log.

BYOK, every request: paste your Anthropic, OpenAI, or LiteLLM-compatible key and DigiChat forwards it to DigiGraph per-request — never stored, never logged. The same Next.js BFF brokers machine API keys for service-to- service calls, so a single deployment serves humans and agents.
