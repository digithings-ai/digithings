                // why digithings
                
Four properties of every module.

            

            

                
01
Self-hosted by default
One docker-compose file runs the whole stack on a laptop, a VM, or a cluster — same image, same config.

                
02
BYOK, every request
Anthropic, OpenAI, or any LiteLLM-compatible key. Forwarded per-request — never persisted, never logged.

                
03
Audit-on by default
Immutable JSONL audit, correlation IDs across every span, PII redacted before logs hit disk.

                
04
Backend-swappable
DigiSearch over Chroma / pgvector / Qdrant. DigiStore over S3 / MinIO / Postgres. Change later, free.
