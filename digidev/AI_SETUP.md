# digidev — AI-assisted setup

Copy either prompt below and paste it into Claude Code (or any coding agent) in your project's root directory. The agent will walk through the full setup interactively.

---

## Option A — Full guided setup (recommended)

Paste this into Claude Code:

```
I want to set up digidev in this repository. digidev is an agentic coding
workflow kit that layers structured tasks, quality gates, and guardrails on
top of the tools I already use — it doesn't replace them.

Please:

1. Read `digidev/AGENT_GUIDE.md` for a full overview of what gets installed.

2. Detect what already exists in this repo:
   - Git platform (look for .github/, .gitlab-ci.yml)
   - Existing issue tracker (Jira/Linear config files, .env vars)
   - Tech stack (pyproject.toml, package.json, Cargo.toml, go.mod)
   - Whether a Makefile already exists
   - Whether .claude/ already exists (and what's in it)

3. Ask me about my existing tools before configuring anything:
   a) "Which git platform do you use? GitHub / GitLab / Bitbucket / Other"
   b) "Which tool do you use for issue management? GitHub Issues / Jira / Linear / Notion / Other"
   c) "Do you use a team communication platform? Slack / Teams / Discord / None"
   d) "Which database are you using, if any? Supabase / PostgreSQL / SQLite / None / Other"
   e) "Which AI coding agents does your team use? Claude Code / Copilot / Cursor / All"

4. Ask me for required project config:
   - Project name (short slug)
   - GitHub/GitLab org or username
   - Repository name
   - Integration branch (usually "develop" or "main")
   - Production branch (usually "main")
   - My GitHub/GitLab handle(s)
   - My project's components/services (name + description + test command each)

5. Run the installer:
      bash digidev/install.sh
   If digidev/digidev.yml doesn't exist, the installer wizard will ask for all
   required values interactively.

6. Complete remaining setup:
   a) Add `include Makefile.digidev` to Makefile (or rename if no Makefile exists)
   b) Run `make hooks-install` to install the git pre-push hook
   c) Read `digidev-mcp-setup.md` (generated by installer) and complete each MCP step
   d) If using GitHub: create the labels printed by the installer
   e) Commit everything:
        git add .claude/ .github/ scripts/ agents.yml AGENTS.md .mcp.json
        git add Makefile.digidev */AGENTS.md docs/ digidev/ digidev-mcp-setup.md
        git commit -m "chore: install digidev agentic workflow kit"
        git push

7. Verify with `make status` — it should list open agent-task issues (or show "none yet").

Important:
- Ask me about existing tools BEFORE running the installer — the wizard needs my answers.
- Do not guess my org name, repo, or component list.
- If I already have a .claude/settings.json, merge the hook entries rather than overwriting.
```

## Option B — Quick install (if you know your setup)

If you already know what tools you use and just want to run the wizard:

```bash
bash digidev/install.sh
```

The wizard will ask each question interactively and create `digidev/digidev.yml`.

```bash
cp digidev/digidev.example.yml digidev/digidev.yml
# Edit digidev/digidev.yml with your project details

```

## After setup — first workflow

```bash
make new-task          # create your first agent-task issue
# → prompts for: title, tier, risk, component, goal, acceptance criteria
# → creates issue in GitHub / Jira / Linear (whichever you configured)

make task ISSUE=1      # start work in an isolated git worktree

# implement + test your changes...

make score             # automated checks + show rubrics
make score SCORES="security=8,quality=8,optimization=7,accuracy=9"
                       # validate self-scores

make commit MSG="feat(api): add /healthz endpoint (#1)"
make pr                # opens draft PR with pre-filled template
```

## MCP servers — what gets configured

The installer generates `.mcp.json` based on your tool choices. These MCP servers are all **off-the-shelf** — installed via npm, no custom code:

| Tool | MCP server |
|---|---|
| GitHub | `@modelcontextprotocol/server-github` |
| GitLab | `@modelcontextprotocol/server-gitlab` |
| Jira | `mcp-atlassian` |
| Linear | `@linear/mcp-server` |
| Slack | `@modelcontextprotocol/server-slack` |
| Notion | `@notionhq/notion-mcp-server` |
| Supabase | `@supabase/mcp-server-supabase` |
| PostgreSQL | `@modelcontextprotocol/server-postgres` |

The `digidev-mcp-setup.md` file (generated by the installer) has the exact commands to install and configure each one.

## What digidev builds on top of

digidev extends the tools you already have in Claude Code — it doesn't replace any of them:

| Claude Code feature | How digidev uses it |
|---|---|
| **PreToolUse hooks** | 9 guard scripts — block unsafe writes, protect sensitive paths, enforce network rules |
| **Subagents** (`.claude/agents/`) | 6 agents: spec-writer, pr-reviewer, component-router, security-reviewer, test-first-implementer, dictation-normalizer |
| **Skills** (`.claude/skills/`) | 6 skills: finish-task, score-and-fix, triage, worktree-task-start, write-acceptance-criteria, ci-triage |
| **Slash commands** (`.claude/commands/`) | `/spec`, `/score`, `/task`, `/triage`, `/normalize` |
| **MCP servers** | Reads/writes your Jira, Linear, Slack, Supabase directly from Claude Code |
| **Git worktrees** | `make task ISSUE=N` creates an isolated worktree per task |
