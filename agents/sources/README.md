# agents/sources/

Authoritative content for every skill, subagent, and slash command used by any coding agent on this repo.

## Why this exists

DigiThings is developed with multiple coding agents in parallel — Claude Code, Cursor, GitHub Copilot, and others. Without a single source of truth, a skill authored for one tool silently drifts from the others. This directory is the *only* place these definitions live in hand-written form.

## Layout

| Source path | Generated destination | Consumed by |
|-------------|----------------------|-------------|
| `agents/sources/subagents/<name>.md` | `.claude/agents/<name>.md` | Claude Code (first-class subagents) |
| `agents/sources/skills/<name>/SKILL.md` | `.claude/skills/<name>/SKILL.md` | Claude Code skills |
| `agents/sources/commands/<name>.md` | `.claude/commands/<name>.md` | Claude Code slash commands (`/<name>`) |

Cursor (`.cursor/rules/digithings.mdc`) and GitHub Copilot (`.github/copilot-instructions.md`) don't support structured subagents/skills/commands. They receive **prose summaries** of every entry — name, purpose, when-to-use — so they know the same playbook.

## Workflow

1. Edit a source file under `agents/sources/`.
2. If adding a new skill/subagent/command, also declare it under `claude_code_surface:` in [`agents.yml`](../../agents.yml).
3. Run `make agents-init` (calls `scripts/agents_init.py`).
4. Commit the source + the regenerated outputs together.

CI catches drift via `scripts/agents_init.py --check` — a PR whose generated files don't match its sources will fail the `Docs` workflow.

## Do not edit the generated files

`.claude/agents/*.md`, `.claude/skills/*/SKILL.md`, `.claude/commands/*.md`, `.cursor/rules/digithings.mdc`, and `.github/copilot-instructions.md` all carry a **"Generated by scripts/agents_init.py"** header. Hand-edits will be overwritten on the next `make agents-init` run.

## Catalogue

### Subagents (`agents/sources/subagents/`)

| Name | Source file | When to invoke |
|------|-------------|----------------|
| `dictation-normalizer` | `dictation-normalizer.md` | Reshape rambling dictated input into a structured block; invoke via `/normalize`. |
| `component-router` | `component-router.md` | Map a described change onto the right component + reading list + test command. |
| `spec-writer` | `spec-writer.md` | Emit issue bodies matching `.github/ISSUE_TEMPLATE/agent_task.yml`; invoke via `/spec`. |
| `pr-reviewer` | `pr-reviewer.md` | Rubric-aware PR review aligned with `docs/scoring/`. |
| `test-first-implementer` | `test-first-implementer.md` | Red/green/refactor TDD loop bound to the component test command. |
| `security-reviewer` | `security-reviewer.md` | Focused OWASP + DigiThings-specific security sweep before PR review. |

### Skills (`agents/sources/skills/`)

| Name | Source file | When to invoke |
|------|-------------|----------------|
| `triage` | `triage/SKILL.md` | Diagnose PR CI failures — fetch checks, bucket by type, emit minimal fix commands. Triggers on "/triage", "why is CI failing". |

### Slash commands (`agents/sources/commands/`)

| Command | Source file | Description |
|---------|-------------|-------------|
| `/normalize` | `normalize.md` | Invoke `dictation-normalizer` subagent. |
| `/spec` | `spec.md` | Invoke `spec-writer` subagent. |
| `/score` | `score.md` | Run the scoring gate via `make score`. |
| `/task` | `task.md` | Start a backlog task via `make task ISSUE=N`. |
| `/triage` | `triage.md` | Triage CI failures for a PR number. |
