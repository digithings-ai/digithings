# agents/sources/

Authoritative content for every skill, subagent, and slash command used by any coding agent on this repo.

## Why this exists

DigiThings is developed with multiple coding agents in parallel — Claude Code, Cursor, GitHub Copilot, and others. Without a single source of truth, a skill authored for one tool silently drifts from the others. This directory is the *only* place these definitions live in hand-written form.

## Layout

| Source path | Generated destination | Consumed by |
|-------------|----------------------|-------------|
| `agents/sources/subagents/<name>.md` | `.claude/agents/<name>.md` | Claude Code (first-class subagents) |
| `agents/sources/skills/<name>/SKILL.md` | `.claude/skills/<name>/SKILL.md` | Claude Code skills |
| `agents/sources/commands/<name>.md` | `.claude/commands/<name>.md` | Claude Code slash commands (`/<name>`) |

Cursor (`.cursor/rules/digithings.mdc`) and GitHub Copilot (`.github/copilot-instructions.md`) don't support structured subagents/skills/commands. They receive **prose summaries** of every entry — name, purpose, when-to-use — so they know the same playbook and can emulate the intent when prompted.

## Workflow

1. Edit a source file under `agents/sources/`.
2. If adding a new skill/subagent/command, also declare it under `claude_code_surface:` in [`agents.yml`](../../agents.yml).
3. Run `make agents-init` (calls `scripts/agents_init.py`).
4. Commit the source + the regenerated outputs together.

CI catches drift via `scripts/agents_init.py --check` — a PR whose generated files don't match its sources will fail the `Docs` workflow.

## Do not edit the generated files

`.claude/agents/*.md`, `.claude/skills/*/SKILL.md`, `.claude/commands/*.md`, `.cursor/rules/digithings.mdc`, and `.github/copilot-instructions.md` all carry a **"Generated by scripts/agents_init.py"** header. Hand-edits will be overwritten on the next `make agents-init` run and will fail CI drift checks.
